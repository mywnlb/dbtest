# Server XA 与持久 XID Registry v1

## 目标

- 在实例根目录新增 `mysql.xa` append-only XID registry。
- Registry frame 使用稳定版本、sequence、CRC32C 和强制 fsync。
- 支持 XA START/BEGIN、END、PREPARE、COMMIT、ROLLBACK、RECOVER SQL。
- SQL/session 继续只持有 opaque transaction handle，不暴露 storage Transaction。
- 把 registry 的恢复决议注入 StorageEngine PREPARED recovery。
- 未决 PREPARED 阻止普通 DatabaseEngine OPEN。
- 提供只取得 instance lock、不开 storage/session 的离线决议工具。

## 关键决策

- XID 由 signed formatId、1..64B gtrid、0..64B bqual 组成，总字节不超过 128。
- Registry 状态固定为 PREPARING、PREPARED、COMMIT_DECIDED、ROLLBACK_DECIDED、COMPLETED。
- PREPARE 顺序为 durable PREPARING -> storage prepare/fsync -> durable PREPARED。
- Phase two 顺序为 durable decision -> storage terminal/fsync -> durable COMPLETED。
- PREPARING 在恢复时只允许推导 ROLLBACK；PREPARED 没有外部决议时 fail-closed。
- COMMIT_DECIDED/ROLLBACK_DECIDED 是不可逆决议，重试不得切换方向。
- Storage 成功完成启动恢复后，消费过的 PREPARING/DECIDED registry entry 才写 COMPLETED。
- Registry 只容忍并截断不完整文件尾；完整 frame 的 magic/version/CRC/状态损坏均 fail-closed。
- XA 只读分支用普通 commit 返回 READ_ONLY，不创建 PREPARED storage owner。
- 活动 XA branch 不支持跨 Session 迁移；同 Session JOIN/RESUME 只重开已 END/SUSPEND 的同一 branch。
- 已 PREPARED branch 放入 DatabaseEngine 共享 coordinator，phase two 可由其它 Session 发起。
- XA ONE PHASE 只处理当前 Session 已 END 的活动 branch，不写持久 decision。
- XA RECOVER 只返回 registry 中未决 PREPARED，不从 undo 页猜测 XID。

## 非目标

- 不实现分布式事务管理器、heuristic decision、跨实例原子提交或网络协议 XA packet。
- 不恢复 crash 前仍 ACTIVE、但尚未完成 storage prepare 的会话对象。
- 不支持活动 branch 跨 Session migrate；FOR MIGRATE 只接受语法并明确拒绝执行。
- 不对 `mysql.xa` 做在线 compaction；COMPLETED frame 作为审计尾保留。

## 验收

- XID 边界、字节防御性复制、等值与文本表示测试通过。
- Registry 覆盖 reopen、状态转换、fsync 后快照、尾部截断与 CRC 损坏拒绝。
- Parser 覆盖全部 XA grammar、JOIN/RESUME/SUSPEND/FOR MIGRATE 与非法尾部。
- Gateway 覆盖有写 prepare、prepared commit/rollback、只读 identity。
- Session 覆盖 active/end/prepare/phase-two、只读、one-phase、recover 与非法状态。
- DatabaseEngine 覆盖 PREPARING 自动 rollback、DECIDED 自动完成、PREPARED 阻止 OPEN。
- 离线 maintenance 覆盖 instance lock 排他和只写 registry 决议。
